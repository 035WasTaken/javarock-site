About JavaRock

JavaRock was started because I believe that Java Edition is superior, yet many of my friends only can play on Bedrock. I found a plugin called GeyserMC, and it allows for crossplay between the two editions (with the server running on Java).

The server runs on Fabric 1.17.1, and the current plugins are:

  • GeyserMC - crossplay
  • Floodgate - crossplay
  • Cave Biomes - terrain
  • Crowmap - utility
  • Simpler Harvest - utility
  • Keep Head Names - utility
  • Mini MOTD - utility
  • SAX (1.17 branch) - utility
  • Styled Player List - utility
  • Lithium - performance
  • Starlight - performance
  • Fat Experiance Orbs - performance
  • Fast Furnace - performance
  • World Specific View Distance - performance

Also, the changed gamerules are:

  • PlayersSleepingPercentage: 25% (yeah thats it lol)

Bedrock and Java Edition crossplay isn't perfect however, so there are a few downsides to playing the server on Bedrock as opposed to Java (which is why if you have the option you should play on Java as these issues don't exist there):

  • On mobile, buckets only place in the middle of the screen as the Java Edition packet doesn't specify where the block is placed
  • Bedrock players may occasionally experiance rubberbanding when going over slabs, stairs, etc. as Bedrock treats their hitboxes differently
  • Bedrock players can't easily put things other than shields, totems, and arrows into their offhands as this is a client-side limitation. You can emote to swap your main hand and offhand, but you can't use your offhand items unless you can on Bedrock.
  • The glowing effect doesn't work and the spectral arrow texture is just a normal arrow as they don't exist in Bedrock
