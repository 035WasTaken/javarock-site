Server Software

JavaRock Season 4 is running on Fabric 1.18.1 with Lithium and Starlight used to increase the performance of the server. (Lithium makes server calculations more efficient and Starlight improves the lighting engine, which is better than Phosphor.)

To allow crossplay, we have GeyserMC and Floodgate, which translate the packets from the Java Edition so that the Bedrock client can understand them, and vice versa. Floodgate converts Bedrock XUIDs to Java UUIDs so that Bedrock players don't need a Java account to play.

Bedrock Info

Since the Bedrock and Java Editions are fundamentally different, if you are playing on Bedrock Edition, some mechanics are different. These include:

  • Combat: If you are an avid Bedrock player, combat works a bit different in the Java Edition. There is an attack cooldown (represented by a bar in the middle of the screen when you attack), and your weapon won't do as much damage if it's not full. Also, axes do more damage than swords, but have a longer attack cooldown.
  • Redstone: Some redstone devices and mechanics (eg. quasi-connectivity) work slightly differently in Java than in Bedrock. If you are building a tutorial for some redstone device, Bedrock ones may not work. This also applies to things such as villager breeders/etc.
  • Raids: You can get up to Bad Omen V (by killing 5 banner pillagers), which will result in a harder, but more rewarding, raid.
  • Movement: Crawling in 1 block tall spaces is possible (via being pushed into a 1 block tall space), and you can sneak into 1.5 block tall gaps.
  • Nether: The nether build limit is now y=256, meaning you can make your gold farms on the nether roof!
  • Offhand: Although you cannot use the item unless it's possible to do so in Bedrock, you do have the ability to put any item in your offhand now through emoting or running '/geyser offhand'.

Bugs

There are a few things that may be buggy for Bedrock players due to the differences in the two games, notably:

  • Mobile Buckets: Java Edition's packet for placing and picking up fluids only includes that the player is doing this, so they only place in the middle of the screen, regardless where you tap. (To remedy this, you can go into Settings > Touch and enable Split Controls.)
  • Crafting items in bulk: Crafting a ton of an item may be a bit slow due to the way Geyser translates crafting packets.
  • Bamboo: Bamboo (and also sometimes dripstone) is extremely buggy when you walk through it. This is because Bedrock renders the bamboo in a different spot than it is on Java, making the server thing you are in the block and lagging you back.
  • Damage: The bedrock attack damage descriptions for weapons are incorrect, since 1.9 combat did not really come to Bedrock.
  • Slabs: Rarely there may be some slab or stair that lags you back when you try to walk up it, this is because Bedrock and Java treat the hitbox of stairs differently.

Settings

There are two settings that you can change for yourself specific to the server; enabling/disabling showing coordinates and setting the attack cooldown indicator to under your crosshair or above your hotbar. These are specific to you and can be accessed through Settings > Server.

Linking

You can link your Java and Bedrock accounts (so that when you join on Bedrock you will be automatically logged into your Java account) by entering the linking server (link.geysermc.org) and running /linkaccount.

Consoles

Console users will be aware that you cannot join custom servers. Or so you thought! There are (simple) ways around this limitation:

Method 1:

Requires: a phone/tablet (connected to the same network as your console), <5 minutes

Works on: any console

This method simply requires you to download an app (Bedrock Together (iOS) and MineProxy - Droid (Android). In the app, you just have to enter the IP of the server you want to join, the port (default is 19132), and tap run or start. The server will then show up as a LAN game.

Method 2:

Requires: the server tab, 4-8 minutes

Works on: XBOX and Nintendo Switch

This method is slightly more complicated, as it requires changing your DNS. Videos on how to do this can be found below, but what's important is that you need to change your primary DNS to 104.238.130.180, and change the secondary DNS to any popular DNS server (I prefer Cloudflare's DNS server (1.1.1.1), although in the videos he uses Google's DNS (8.8.8.8).)

To undo this, you can change your DNS back to automatic.
